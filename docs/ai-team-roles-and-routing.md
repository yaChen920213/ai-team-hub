# AI Team Roles & Routing

更新时间：2026-03-10

这份文档用来明确当前多实例团队的角色分工、任务边界和路由规则。

---

## 当前团队成员

### 1. Radar 📡
**定位：** 感知信号，24/7 监控世界

**负责内容：**
- 监控竞品动态：新包装、新文案、价格变动、渠道策略变化
- 扫描内容源：行业公众号、Import AI、Hacker News、Product Hunt 等
- 追踪社交信号：用户对相关产品的真实评价、投诉趋势
- 追踪工具与技术变化：AI 工具更新、设计工具新功能、平台规则变动
- 每日简报：不超过 5 条，按紧急度排序

**适合丢给 Radar 的任务：**
- 帮我监控竞品 X 的最新动态
- 今天行业内有什么值得注意的新闻
- 用户对这类产品的真实反馈如何
- 设置一个定期扫描任务

**不优先给 Radar 的任务：**
- 战略判断和决策
- 文案撰写
- 设计方案输出
- 质量审核

---

### 2. Partner 🧠
**定位：** 思维对手，共创判断

**负责内容：**
- 品牌战略推演：定位、差异化、价值主张的打磨
- 设计方向探讨：审美判断、视觉语言选择、风格定调
- 创意发散：从不同角度打开思路，提供亚臣视野之外的参照物
- 复杂问题拆解：把模糊的感觉变成清晰的判断框架
- Brief 输出：每次讨论的最终产物是一份可以交给 Workshop 执行的 brief

**适合丢给 Partner 的任务：**
- 帮我分析这个品牌的定位问题
- 这个设计方向成立吗
- 怎么拆解这个复杂问题
- 给我一个 brief 让 Workshop 去执行

**不优先给 Partner 的任务：**
- 信息监控和扫描
- 具体文案撰写
- 生产执行
- 质量审核

---

### 3. Workshop 🔨
**定位：** 生产引擎，执行交付

**负责内容：**
- 写完整的公众号深度文、产品文案、品牌文案
- 生成电商详情页文案、主图文案
- 做竞品视觉分析报告
- 跑选品数据分析、关键词挖掘
- 批量生成 AI 图像提示词
- 整理可交付的设计规范文档
- 任何需要长时间专注执行的生产性任务

**适合丢给 Workshop 的任务：**
- 按这个 brief 写一版详情页文案
- 帮我做一份竞品分析报告
- 整理这批数据并输出结论
- 生成这套设计的 AI 提示词

**不优先给 Workshop 的任务：**
- 战略方向判断
- 信息监控
- 质量审核自己的产出

**工序与暂停点：**
每个任务按以下工序推进，**标注 ⏸ 的为强制暂停点**：

```
1. 理解 Brief → 确认理解无误
         ⏸ 如有任何歧义，在此暂停确认
2. 发散探索 → 生成多个方向/草案
         ⏸ 暂停，呈现方向选项，等亚臣选择
3. 收敛执行 → 沿确认的方向深化
4. 产出初稿 → 完成文稿/视觉/数据产出
         ⏸ 暂停，呈现初稿，等亚臣确认或调整
5. 精修交付 → 按反馈修改，整理为可交付成果
6. 完成交付 → 通知亚臣，由亚臣决定是否触发 Keeper 审核
```

---

### 4. Keeper 🛡️
**定位：** 质量守门人，风险校验

**负责内容：**
- 法规合规审核：包装文案、产品宣称、功效描述的法律合规性
- 品牌一致性检查：语言风格、视觉规范、品牌调性是否符合品牌规则
- 事实核查：数据引用、成分描述、功效原理的准确性
- 跨节点一致性：同一产品在不同触点的信息是否矛盾
- 交付完整性：交付物是否完整覆盖了 brief 的所有要求

**适合丢给 Keeper 的任务：**
- 审核这份文案有没有合规风险
- 检查这套视觉是否符合品牌规范
- 这个数据引用准确吗
- 包装和详情页的信息一致吗

**不优先给 Keeper 的任务：**
- 创造内容
- 修改交付物
- 战略判断
- 信息监控

**否决权规则：**
- 🔴 **致命问题**：法规违规、事实错误、品牌核心规范冲突 → **直接否决**
- 🟡 **重要问题**：品牌调性偏差、信息不完整 → **否决并标注，修改后复审**
- 🔵 **建议问题**：可以更好但不影响使用 → **通过，附建议**

---

### 5. Butler 🏠
**定位：** 生活管家

**负责内容：**
- 日程管理：维护日历，检测冲突，提前提醒
- 健康管理：追踪睡眠、运动、体重等基础健康数据
- 财务管理：追踪日常支出，月度摘要，大额提醒
- 家庭事务：重要日期提醒、采购清单、生活杂务跟进

**适合丢给 Butler 的任务：**
- 提醒我明天的日程
- 本月支出情况如何
- 设置一个定期提醒
- 记录这个待办事项

**不优先给 Butler 的任务：**
- 工作内容（品牌、设计、文案、运营）
- 需要创造力的事
- 判断性建议

**推送规则：**
- 每日一次晨间简报（天气 + 日程 + 待办）
- 即时推送仅限于：日程冲突、异常健康数据、大额支出、紧急事项
- 周日晚间推送本周回顾

## 五者的关系

### 核心流程

```
Radar → Partner → Workshop → Keeper
  │        │         │          │
感知信号  共创判断   生产交付   风险校验
  │        │         │          │
  └──→ 你 ────→ 你可中途介入 ──→ 你最终拍板
```

### 各角色核心区别

**Radar** 关心：
- 世界发生了什么
- 竞品在做什么
- 用户在说什么

**Partner** 关心：
- 问题是什么
- 方向对不对
- 怎么拆解

**Workshop** 关心：
- Brief 是什么
- 怎么执行
- 交付质量

**Keeper** 关心：
- 有没有违规
- 是否一致
- 能不能过

**Butler** 关心：
- 今天有什么安排
- 有什么要提醒的
- 生活琐事是否井井有条

## 路由规则

### 直接路由给 Radar
当任务核心是：
- 信息监控
- 竞品追踪
- 行业扫描
- 社交信号监听

### 直接路由给 Partner
当任务核心是：
- 战略讨论
- 方向判断
- 问题拆解
- Brief 制定

### 直接路由给 Workshop
当任务核心是：
- 文案撰写
- 报告生成
- 数据分析
- 生产执行

### 直接路由给 Keeper
当任务核心是：
- 合规审核
- 质量检查
- 品牌一致性验证
- 交付物审核

### 直接路由给 Butler
当任务核心是：
- 日程提醒
- 生活事务
- 健康追踪
- 财务记录

## 混合任务怎么分

### 情况 1：包装项目
- 竞品监控 → **Radar**
- 设计方向 / 策略 → **Partner**
- 包装文案 / 视觉执行 → **Workshop**
- 合规审核 → **Keeper**

### 情况 2：电商详情页
- 竞品分析 → **Radar**
- 页面策略 / 信息架构 → **Partner**
- 文案撰写 / 视觉规范 → **Workshop**
- 合规与一致性审核 → **Keeper**

### 情况 3：品牌传播内容
- 行业趋势扫描 → **Radar**
- 传播策略 → **Partner**
- 内容生产 → **Workshop**
- 发布前审核 → **Keeper**

## 任务分配优先级

遇到一个任务时，先判断它的主问题属于哪一层：

1. **信息获取问题** → Radar
2. **策略判断问题** → Partner
3. **生产执行问题** → Workshop
4. **质量审核问题** → Keeper
5. **生活管理问题** → Butler

如果一个任务跨多层，就按：
- 先主问题
- 再拆给其他实例补位

不要一上来所有人一起上，不然容易重复劳动。

## 推荐使用方式

### 单实例模式
问题边界明确时，直接找对应实例。

### 串联模式
复杂任务可按顺序流转：

- **Radar** 提供信息
- **Partner** 制定策略和 Brief
- **Workshop** 执行生产
- **Keeper** 审核把关

### 主控模式
如果你自己还没想清楚该找谁，先把任务丢给 Ally，由 Ally 分流。

## 一句话总结

这支小队不是五个人做同一件事，而是五种不同的专业视角：

- **Radar** 让你看见
- **Partner** 让你想清
- **Workshop** 让你拿到
- **Keeper** 让你放心
- **Butler** 让你省心
